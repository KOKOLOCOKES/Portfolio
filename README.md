## 1. 로봇 암 기반 3D 스캐닝을 통한 스마트 양식 객체 디지털 트윈 시스템 개발 (캡스톤 디자인)


- 개발 기간: 2025.04 - 2025.06
- 역할
    - RGB-D 카메라를 사용하여 3D 맵 제작 서칭 및 연구
    - ULIP 임베딩 기반의 질의-객체 매칭
    - LLM API 연동

- 어려웠던 점/해결방안
    - 로봇 암으로 촬영해 생성한 3D 공간 데이터는 2D 이미지-텍스트 매칭 모델인 CLIP에서 사용 불가
        1. 3D 공간 데이터에서 물체를 Segmentation 모델로 감지한 뒤 마스크를 씌워 해당 부분만 잘라내는 방식으로 2D 이미지를 입력으로 사용했습니다.
        2. 3D Point cloud를 함께 매칭하는 ULIP 모델을 적용해 보았습니다.
    
    - ULIP 모델 학습을 위한 방대한 데이터 수집  및 학습
        1. Pretrained model이 없었으며, 논문에서 제시한 학습 데이터가 접근 불가능한 경우가 많아 데이터 수집 및 검증에 많은 시간 소요됐습니다.
        2. 방대한 데이터를 전체 학습 시키기엔 컴퓨터 성능 및 물리적 시간이 부족하다는 문제가 발생했습니다.
        3. 객체당 데이터 갯수를 제한시키고 학습을 진행하였으나 정확도 저하되는 문제가 있었습니다.
    
    - 프로젝트 기간 중  ULIP github가 업데이트 되어 Pretrained model을 사용할 수 있게 되어 시연에 사용할 물체에 대해 파인튜닝을 진행
        1. ULIP 학습에 사용된 데이터와 파인튜닝을 위한 데이터의 밸런스가 맞지않아 학습이 정상적으로 진행되지 않는다고 판단했습니다.
        2. 데이터 Augmentation 기법을 사용하여 데이터를 일부 추가하여 학습을 진행했습니다.

## 3. Ms.Pacman 이미지 기반 학습 연구


- 개발 기간: 2022.06 - 2022.10
- 학부 인턴 개인 과제

- 어려웠던 점/해결방안
    - 온전한 게임 이미지를 사용할 수 없는 문제
        1. Ms.Pacman이 비정기적으로 깜빡여 게임 이미지 자체를 학습에 사용할 수 없었습니다.
        2. 캐릭터 좌표값을 기반으로 새롭게 맵을 제작하여 학습에 사용했습니다.
    
    - Ms.Pacman의 입력 유지 문제
        1. Ms.Pacman은  다음 이동키가 입력될 때까지 같은 방향으로 전진하는 특징이 있었습니다.
        2. 모델 추론 시간 동안 캐릭터가 이동하여 실제 위치가 맞지 않아 정상적으로 이동하지 못하는 문제가 있었습니다.
        3. 추론 속도 때문에 완전 실시간이 어렵다고 생각하여 맵 제작 속도를 높여 간격을 줄이고자 C++ 버전의 OpenCV를 사용하여 구현하고자 하였으나 여러 문제가 있어 실제로 적용하지 못한 것이 아쉽습니다.

## 4. Ice Shooting - 하이퍼 캐주얼 게임 개발


- 개발 기간: 2022.01 - 2022.02
- 2인 팀프로젝트

- 어려웠던 점/해결방안
    - 처음 접해보는 C#과 Unity
        - C#과 Unity에 대해 전혀 알지 못하는 상태로 급하게 시작된 프로젝트였기에 개발과 동시에 공부를 해야 했습니다.
        - 개발 중 공부한 내용은 따로 정리하여 공통적으로 도움이 될 만한 정보를 취합해 영상을 촬영하였으며, 대학 내 사이버 강의실에 모두가 볼 수 있게 공유 했습니다.
        
    - 개발 방향에 대한 팀원과의 의사소통 문제
        - 개발 도중 새로운 기능 추가 또는 조작 편의성 등 여러 부분에서 의견이 오갔으며, 그 과정에서 시간 제약 및 기술적인 한계 때문에 여러 문제가 발생했습니다.
        - 서로를 설득하기 위해 주장에 대한 자료를 정리하고 대화하는 방향으로 프로젝트를 진행하였으며, 의견이 좁혀지지 않을 때 교수님께 양해를 구하고 중재를 요청 드렸습니다.
